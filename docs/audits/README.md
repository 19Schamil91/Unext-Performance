# Audit-Artefakte

`docs/audits/` enthaelt Audit-, Review- und Pruef-Artefakte.

Screenshots in diesem Ordner sind keine Website-Assets. Website-Assets gehoeren nach `public/`.

Neue Audit-Screenshots sollen nicht im Projektroot abgelegt werden.

Logs und lokaler Tool-State sollen nicht committed werden.

`screenshots/root/` enthaelt ehemals im Root liegende Audit-Screenshots.

`screenshots/codex-runs/` enthaelt ehemals in `.codex-screenshots/` liegende Codex- und Audit-Screenshots.
