# Aufgabe 002: Workflow-Struktur anlegen

## Ziel

Den Ordner `workflow/` mit `todo/`, `done/` und einer kurzen README anlegen.

## Kontext

Geplante Aufgaben sollen sichtbar in `workflow/todo/` liegen. Abgeschlossene Aufgaben duerfen erst nach ausdruecklicher Freigabe nach `workflow/done/` verschoben werden.

## Vorgehen

- `workflow/todo/` anlegen.
- `workflow/done/` anlegen.
- `workflow/README.md` mit der Workflow-Regel erstellen.

## Akzeptanzkriterien

- Die Workflow-Ordner sind vorhanden.
- Die README beschreibt SPEC -> PLAN -> CODE -> REVIEW -> DONE.
- Die README enthaelt die Pflichtfrage vor dem Verschieben nach `done/`.
